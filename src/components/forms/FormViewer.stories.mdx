# Form Viewer JSON Specification

## Object model skeleton

The form viewer expects a JSON with the following structure.

```json
{
  "id": 1234567890
  "internalTitleEn": "",
  "internalTitleFr": "",
  "publishingStatus": true,

  "form":
    {
      "version": 1,
      "titleEn": "",
      "titleFr": "",
      "submission": {
        "templateID": "",
        "email": ""
      },
      "questions":[ ... ]
    }
  }


```

There are 3 main parts to the structure:

1. Form object in the context of Form Builder

   `id`: The unique id (UUID) for the form (`int`)

   `internalTitleEn/Fr`: Sets the title as seen from the Form Builder application (`string`)

   `publishingStatus`: Set from the Form Builder as to whether this form can be rendered (`bool`)

2. Form object in the context of Form Viewer

   `version`: Defines the static version of the form (`int`)

   `titleEn/Fr`: The displayed title of the form to the user (`string`)

   `submission`: An options object that defines how the form is submitted. (`object`)

   ​ -> `templateID`: Defines which Notify template to use (`string`)

   ​ -> `email`: Which email addess will the form be submitted to through Notify (`string`)

   `questions`: An array of question objects (`array`)
